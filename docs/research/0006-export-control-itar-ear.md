**Status:** 学習用メモ (Draft)

## なぜ押さえるか

- 航空・防衛系材料を扱う案件では **輸出管理**が Matlens の
  アーキテクチャ（クラウド可否・アクセス制御・国籍判定）を直接制約する
- 「知らずに」海外クラウドに機密データを置くと、
  技術データの **みなし輸出**（非居住者へのアクセス = 輸出）に該当し得る
- 罰則は刑事罰（懲役・罰金）+ 輸出実績の剥奪（事業継続に直結）

---

## 3 つの法域

### 1. ITAR (International Traffic in Arms Regulations) — 米国防省

- 対象: **軍用品リスト (USML)** — 軍用機・ミサイル・軍艦・核・衛星等
- 主管: 米国務省 DDTC (Directorate of Defense Trade Controls)
- 対象材料例: **軍用エンジン部品 / 宇宙ロケット関連材料**
- 原則: **非米国人への開示禁止**（米国市民権 or 永住権 or 特定保護ステータスのみ）
- 該当製品の **技術データ**（図面・試験結果・レシピ）は
  グローバルクラウドに置けない（米国リージョン限定でも ID 認証で米国人のみに制限が必要）

### 2. EAR (Export Administration Regulations) — 米国商務省

- 対象: **両用品目** (dual-use) — 民生利用できるが軍事転用可能なもの
- 主管: 米国商務省 BIS (Bureau of Industry and Security)
- 対象材料例: **高性能セラミクス / 耐熱合金 / 炭素繊維 / 高精度計測機器**
- ECCN (Export Control Classification Number) で分類
  - 例: **ECCN 1C002** (特殊合金), **1C010** (炭素繊維)
- 再輸出規制あり（米国原産品は世界中で米国法の制約を受ける）

### 3. 外為法（外国為替及び外国貿易法） — 日本

- 主管: 経産省・貿易管理部
- **リスト規制**: 輸出貿易管理令別表第 1（輸出）+ 外国為替令別表（技術）
- **キャッチオール規制**: 別表に載っていなくても「大量破壊兵器転用の懸念」があれば
  許可申請が必要
- **みなし輸出**: 非居住者への技術提供も輸出扱い
- **リスト規制違反の罰則**: 5 年以下の懲役 or 700 万円以下の罰金 or 併科

## 技術データの「輸出」とは

輸出管理でいう「輸出」は物理的な国境越えだけではない:

| 形態 | 該当例 |
|---|---|
| 物理輸送 | 試験片・部品を海外に送る |
| 電子伝送 | メール添付・クラウド API 経由 |
| **みなし輸出 (deemed export)** | 日本国内で非居住者（外国籍従業員・見学者）に見せる |
| クラウドストレージ経由 | データが海外サーバを通過 |
| リモートアクセス | 海外拠点から日本のサーバにログイン |

## Matlens アーキテクチャへの示唆

### クラウド可否の判断フロー（顧客ヒアリング用）

```
Q1. 扱う材料・部品は ITAR 該当？
    Yes → 米国クラウドのみ + 米国人アクセス制限
    No  → Q2
Q2. 扱う材料・部品は EAR 該当？
    Yes → ECCN 別の制約を法務確認
    No  → Q3
Q3. 外為法のリスト規制該当？
    Yes → 輸出許可申請 + データ所在地確認
    No  → キャッチオール該否の自主判断
```

### 対応パターン

| パターン | 技術要件 | 代表構成 |
|---|---|---|
| **フルクラウド OK** | 一般民生案件 | Vercel + Supabase + MSW (既存構成) |
| **国内クラウド限定** | EAR 該当 / 客先ポリシー | AWS Tokyo / さくらのクラウド |
| **オンプレ必須** | ITAR / 社内ポリシー | Docker + Postgres + 社内サーバ |
| **エアギャップ** | 軍用機密 | スタンドアロン exe / USB 配布 |

Matlens は Phase 1 で **Repository パターン + `VITE_BACKEND_MODE` 切替**
を採用しているので、REST を止めて in-memory / ローカル Postgres に
差し替える経路が技術的にある（ADR-001）。オンプレパターンで提案可能。

### アクセス制御の必要項目（ITAR / EAR の場合）

- ユーザの **国籍** を明示的に保持（IdP 側で属性化）
- ログイン時・操作時の **監査ログ**（誰が・いつ・何を）を改ざん防止で保存
- データエクスポート時に **該否判定フラグ**を必須入力
- 画面上で **Restricted バッジ** を常時表示
- PDF / MaiML エクスポートにウォーターマーク

## クラウドサービス側の契約観点

- **Vercel**: 主要リージョンは米国・EU・アジア。データ所在地を指定できる
  機能はプラン依存。EAR 該当データは **要検討**
- **Supabase**: Pro プランで region 指定可能（Tokyo / Frankfurt 等）
- **AWS**: 日本リージョン（ap-northeast-1/3）あり。データ移動の明示契約が可能
- **Azure**: 東日本 / 西日本リージョンあり。政府クラウドは Azure for Government 別契約

## 日本ラボでの実務（仮説）

- 海外 OEM 案件 = **EAR / ITAR 該当の可能性を常に確認**
- 航空系は大半が EAR 9A991（民間航空機部品）該当 — ECCN 別の再輸出可否を要確認
- 国内防衛案件は外為法リスト規制が効く
- 顧客（OEM）は通常 **輸出管理プログラム (EMS / ECP)** を持っており、
  その枠組みで受託先のデータ取扱が指定される

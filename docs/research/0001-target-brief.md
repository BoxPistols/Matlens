### 1.2 本件の立ち位置

- **契約主体は S社** (プラント・インフラ・産業設備向けのエンジニアリング+システム開発子会社)。バックエンドとインフラは S社 内部チームが担当。

## 2. M社グループの材料研究スコープ (公開情報ベース)

### 2.1 M社 Y拠点 の R&D 構成 (推定)

- M社 Y拠点 = M社全社の R&D 中枢の一つ。数百名規模のエンジニア・研究者。
- 材料研究部門は**基盤技術系の7部門**の一つに位置付けられる。2024年に組織改編があった模様。
- 近年**5階建ての統合実験棟**が稼働し、フロア単位で学問領域ごとに集約されている (1フロアが材料・物理研究専用)。
- M社 の完成品製造 (大型の鋳造・鍛造・AM 生産) は別サイトで行われ、**Y拠点 は R&D の頭脳** と位置付けられる。

### 2.2 主要材料カテゴリ

| カテゴリ | 代表材料 (一般化) | 用途 (一般化) |
|---|---|---|
| Ni 基超合金 (DS/SC 鋳造) | 方向凝固・単結晶超合金一般 | 高温タービンブレード・ディスク |
| Ni 基超合金 (AM) | 析出硬化型 Ni 基合金 | AM 製高温部品 |
| γ-TiAl | 金属間化合物系軽量合金 | 低圧タービン軽量部品 |
| SiC/SiC CMC | 3D 織 + CVI/MI + 耐環境コーティング | 1400℃級高温セラミック部品 |
| Co 基超合金 | 真空溶解材 | タービン部品 |
| AM 粉末 | Ti 系・Ni 系金属粉末 | E-PBF / DED 用 |
| 鋼材・ボイラ合金 | Cr-Mo 系耐熱鋼、オーステナイト系耐熱鋼 等 | パワー/産業機械系 |

### 2.3 典型的なキャラクタリゼーション・スタック (推定)

- **顕微鏡**: 光学金相、FE-SEM + EDS、**EBSD**、TEM
- **回折**: XRD (相同定・残留応力)
- **機械試験**: 引張 (RT/高温)、硬度 (Vickers 中心)、**クリープ・クリープ破断** (週〜月単位の長時間試験)、LCF/HCF 疲労、Charpy 衝撃
- **熱分析**: DSC、TGA、ダイラトメータ
- **化学**: ICP-OES、GDS、OES、O/N/H ガス分析 (Ti/AM 用)
- **NDT**: X線CT (AM ポロシティ評価)、UT
- **プロセス側**: 粉末 PSD、流動性、タップ密度

### 2.4 サンプル・ライフサイクル (11工程)

Petri net 可視化の骨格になる:

```
1. プログラム計画
2. 材料選定
3. 原料準備              (インゴット or 粉末 QC)
4. 一次加工 [並列]        (鋳造 DS/SC | 鍛造 | PBF/EBM/DED | CVI/MI)
5. 後加工                (HIP, 熱処理, 機械加工, コーティング)
6. 試験片採取
7. 評価 [並列 fan-out]
   ├── 組織 (OM/SEM-EDS/EBSD/TEM)
   ├── 化学 (ICP/OES/ガス)
   ├── 回折 (XRD/残留応力)
   └── NDT (CT/UT)
8. 機械試験              (引張 → 硬度 → 疲労 → ★クリープ)
9. 環境試験              (酸化, 腐食, 熱サイクル)
10. 破面解析
11. レポート・フィードバック
```

**Petri net モデリング上の決定的な性質:**

1. **工程 4, 7 は fan-out** (同じ親サンプルから複数子トークン)
2. **工程 8 のクリープ試験は place を長時間占有** (数週〜数ヶ月)
3. **工程 10 → 11 → 上流への feedback loop** (再加工・再試験) — **DAG ではなく Petri net を選ぶ決定的理由**
4. **各 transition = 測定イベント に MaiML ファイルをアタッチ**

## 3. データ標準の立ち位置

### 3.1 MaiML (JIS K 0200:2024) の採用状況

公開情報で M社・L社 とも MaiML 採用を明言していない。JIS K 0200:2024 は 2026 年時点でまだ新しい規格 (2024年5月公開)。

**推測**: 評価フェーズと考えられる。Matlens が「MaiML ネイティブな研究データ管理 + ワークフロー可視化」を提案することで、**「最初の MaiML ネイティブ LIMS」としてのポジショニング** が可能。

### 3.2 Matlens 側の既存資産

現状の `src/services/maiml.ts`:

- 完全 round-trip serialize/parse (JIS K 0200:2024 準拠)
- XXE / Billion Laughs 対策済み (DOCTYPE 拒否、10MB 上限)
- テストスイート整備済み
- エクスポートモーダルの primary 選択肢

**本番品質で実装済み。** 足りないのは:

- MaiML ファイルのドラッグ&ドロップインポート動線
- 装置ベンダ固有フォーマット (XRD 系, 顕微鏡系, 試験機系) → MaiML 変換レイヤ
- レガシー Excel / CSV 取り込みの UI

### 3.3 PNML + MaiML の合成

- **PNML (ISO/IEC 15909-2)** = ワークフロー構造の標準 XML
- **MaiML (JIS K 0200:2024)** = 測定結果の標準 XML
- 両者は直交し、以下のように合成可能:

```
PNML 文書 (workflow structure)
├── Place: "熱処理 待機"
├── Transition: "XRD 測定" → artifact: sample-0042-xrd.maiml
├── Transition: "引張試験" → artifact: sample-0042-tensile.maiml
└── ...
```

**実装方針**: Matlens 独自スキーマで workflow を保持 → エクスポート時に PNML 生成 → 測定結果は MaiML で添付。**両標準に対応する初の研究データ製品** を目指す。

## 5. 製品の拡張ビジョン — シミュレーション + 最適化

本プロジェクトは**測定データ管理だけでなく**、研究者がプラットフォーム内で**仮説検証型の in silico 実験**を回せることを目指す。これは競合 LIMS との差別化要素であり、研究者にとっての価値の核心。

### 5.1 シミュレーション機能 (段階的)

#### Tier 1: 組成 → 物性の経験式予測 (v1 で十分実装可能)

- **組成から密度・弾性率を CALPHAD 風の近似式で予測** (Rule of Mixtures, ROM → ROM-VRH)
- **ROM + Hall-Petch** による粒径依存の強度予測
- **Larson-Miller** パラメータでクリープ余命予測
- **Matthiessen's rule** で電気抵抗予測
- これらは数式ベースで数十行のロジックで実装可能 → 研究者が「ちょっと動かせる」体験を提供

#### Tier 2: 可視化中心の現象シミュレーション (v2 目標)

- **熱処理シミュレーション**: 時効析出を単純な JMAK 式で描画 (硬さ vs 時間曲線)
- **環境試験シミュレーション**: 酸化膜成長 (放物則)、腐食減肉、熱応力サイクル
- **破壊力学**: K-N 曲線、Paris 則での疲労き裂進展のインタラクティブ可視化
- **加圧・融解**: 平衡状態図の概念可視化 (実装は経験則式 + 既知相境界の描画)

これらは**本格的な FEM/CALPHAD ソルバではなく、Explorative visualization として**位置付ける。研究者の直感と経験則の間を埋める。

#### Tier 3: ベイズ最適化 (v2〜v3 目標)

研究者が「次にどの条件を試すべきか?」を決める際の支援:

- **獲得関数ベースの次実験提案** (Expected Improvement / UCB)
- **GP 回帰** で既存測定点から特性関数を推定
- **制約付き最適化** (コスト・熱処理時間・組成範囲の制約下で UTS 最大化)
- 実装には `@tensorflow/tfjs` + 自前 GP 実装、または Python Vercel Function 経由 (BoTorch ラッパー)

ユーザー体験としては:

- 既存サンプルのデータポイントを散布図で表示
- 「次の組成候補」を獲得関数のコンター上にハイライト
- 「なぜこれを提案したか?」をサンプル点数・分散・Pareto 境界で説明

### 5.2 シミュレーションの UX 位置付け

**重要**: シミュレーション結果は常に「経験則 / 近似モデルによる参考値」と明示する。本気の設計計算には使われないことを front-load する。

- 各シミュレーション結果には **Tier 表示**: "粗見積もり (ROM)" / "参考値 (JMAK近似)" / "探索支援 (GP推定)"
- **provenance = 'simulation'** として記録 → 実測値と混ざらないよう区別
- 研究者の**黒箱への抵抗** を避けるため、**モデル式を全て表示** する (折りたたみパネルで)

### 5.3 シミュレーションが差別化になる根拠

既存の LIMS / ELN (Benchling, LabVantage, openBIS, NOMAD 等) は **データ管理とプロベナンス** に特化しているが、**インタラクティブなシミュレーション機能を持つものはほぼない**。Matlens がこれを取り込むことで:

1. 研究者の**日常の数値計算** (Excel での手計算) を取り込める
2. **新人研究者の教育ツール** としての価値 (組成いじりで即時反応)
3. **ベイズ最適化** は実データが溜まってからの「次の一手」提案として、LIMS 本体の価値を拡張する
4. 競合との差別化: 「データを貯める」だけでなく「データを使って考える」場所

## 9. リスクと制約

### 9.1 技術リスク

- **PNML 互換性**: CPN セマンティクスを自前実装し、export のみ PNML にする方針。CPN Tools で読めるかの検証が必要。
- **React Flow パフォーマンス**: 40ノードは問題なし。将来 1000+ トークン想定時に viewport culling 必要。
- **MaiML スキーマの将来変更**: JIS K 0200 は改定される。2024 版に fix し、将来版は別ハンドラで対応。
- **シミュレーション精度**: 経験則モデルなので「参考値」「探索支援」として明示。設計計算には使えないことを front-load。

### 9.2 商業・運用リスク

- **クラウド許容性**: 航空宇宙関連は IT セキュリティが厳しい。Vercel が不可になる可能性 → オンプレ版 (Docker + PostgreSQL) 移植プランを早期に準備。
- **既存 LIMS との競合**: 既に商用 LIMS 導入済みの場合、置換ではなく**連携レイヤ** ポジショニングへ転換。
- **MaiML 採用のタイミング**: 未採用なら二重意思決定。移行パス設計が必須。
- **二本立てハーフコミット**: 別案件と並行するため、週次定例・仕様凍結タイミング・ステータス共有のリズムを契約前に明確化する。

### 9.3 情報・体制リスク

- 公開情報ベースの仮説で構成。実データ・実測定装置名・実運用フローへの擦り合わせが必要。
- バックエンド / インフラは S社 内部チーム担当 → **API 仕様凍結のタイミング** と **コンポーネント境界** を早期に握る。
- 設計ドキュメント (ADR, 画面定義書) の置き場所 (Figma / Notion / 社内 Wiki) を初回スプリントで決める。

## Appendix: 参考技術

- **MaiML**: JIS K 0200:2024、XML ベース測定結果形式
- **PNML**: ISO/IEC 15909-2、XML ベース Petri net 形式
- **React Flow** (`@xyflow/react`): ノードベース UI ライブラリ
- **dagre**: 自動レイアウトアルゴリズム
- **Colored Petri Net**: トークンに型付き属性を持たせた拡張 Petri net
- **CALPHAD / Rule of Mixtures**: 物性予測の近似手法
- **JMAK (Johnson-Mehl-Avrami-Kolmogorov)**: 相変態速度論
- **Larson-Miller Parameter**: クリープ余命予測式
- **Gaussian Process Regression**: ベイズ最適化の基礎となる確率的回帰
- **Expected Improvement / UCB**: ベイズ最適化の獲得関数

# 現場入り準備キット

本ディレクトリは、Matlens PoC を **現場（クライアント先）に持ち込んでドメイン議論を駆動する** ための資材集です。
画面を見せる → 仮説をぶつける → 違いを聞き出す、というループを回すことを目的としています。

## 構成

| ファイル | 用途 | 使うタイミング |
|---|---|---|
| [hearing-sheets.md](./hearing-sheets.md) | 画面別ヒアリングシート | 各画面を見せながら段階的に質問 |
| [personas-workflows.md](./personas-workflows.md) | ペルソナ別ワークフロー仮説 | キックオフ前の自己整理、初回 1on1 で「こう想像してます、合ってます？」と振る |
| [tool-comparison.md](./tool-comparison.md) | 既存ツール比較マトリクス | 「今は何使ってますか？」を構造化して聞くため |
| [pain-points.md](./pain-points.md) | pain-point 候補 30 件 | 「これって痛みになってます？」と仮説をぶつける |
| [gaps.md](./gaps.md) | 弱点インベントリ | 自分が知らない領域の事前学習 / 質問リスト |

## 想定した使い方

1. **初回キックオフ前**: `personas-workflows.md` と `gaps.md` を読み込み、自己理解を整える
2. **キックオフ**: 概要説明後、`hearing-sheets.md` の各画面で「想定→質問→反応」を進める
3. **2nd ミーティング**: `pain-points.md` から 5〜10 件選び「これ刺さってますか？」と検証
4. **3rd 以降**: `tool-comparison.md` で「Matlens がフィットする領域」を逆算

## 更新方針

- 現場で得た情報は **`hearing-notes/` ディレクトリ** に日付ファイルで蓄積（このリポではなく社内別レポ推奨、機密性次第）
- 仮説が外れたら本ファイルを **そのまま書き換える**（仮説保持より修正の速さを優先）
- ADR (`docs/adr/`) に判断ログとして残すべきものは別ファイル化

# Pain-Point 候補カタログ

Matlens PoC が「これを解いた」と仮定する痛み 30 件。現場で 5〜10 件選んで「これ刺さってますか？」と打診する。
**仮説検証用なので、外れて当然**。当たり / 外れ / 想定外の痛みを区別する。

凡例:
- 🔥 顕在痛 — ユーザが日常的に意識
- 💧 潜在痛 — 実は痛いが慣れて麻痺
- 🌫 想像痛 — こちらが勝手に想像してるだけかも

## ヒアリング戦術

### 初回（30 分）
- A1, A2, A3, B1, B2 を口頭で振る → 反応が良いものを深掘り

### 2 回目（60 分）
- 1 回目で当たった痛みについて、Matlens の該当画面を実際に開いて見せる
- 反応が薄かった痛みは「実はこう運用してます」を聞き出す

### 3 回目（60 分）
- B5, C1, D1 のような「想像痛」を確認 → 想定外の痛みが出てくることが多い
- 提案フェーズへの橋渡し材料にする

---

## 痛み発見後のアクション

1. **顕在痛 (🔥) が当たった** → ROI 試算可能、提案資料の主軸へ
2. **潜在痛 (💧) が当たった** → 「これ実は時間かかってますよね」と気づかせる導線設計
3. **想像痛 (🌫) が外れた** → 削除、別の仮説に差し替え
4. **想定外の痛み** → 本ファイルに新規追加、優先度を再評価

# 既存ツール比較マトリクス

「今は何使ってますか？」を構造化して聞き出すための仮説マトリクス。
ヒアリング後はこの表を **「実際に使われているもの」「使われていないもの」「Matlens がフィットする領域」** に塗り分けて、提案資料化する。

---

## 想定される既存ツール群

### 商用 LIMS（Laboratory Information Management System）

| 製品 | 提供 | 強み | 弱み | Matlens との位置 |
|---|---|---|---|---|
| LabWare LIMS | LabWare 社 | 大規模 / 法規制対応強 | UI が古い / カスタマイズ高コスト | 補完（フロントエンド改善） |
| Matrix Gemini | Autoscribe 社 | 中堅向け / 柔軟 | 切削系特化機能なし | 補完（切削ドメイン追加）|
| Sample Manager LIMS | Thermo Fisher | 化学系強 | 物理試験は弱め | 補完（材料試験フロー） |
| 社内開発 LIMS | 自社内製 | 業務密着 | 改修速度遅 / 属人化 | 置換 or 補完 |

### 汎用ツール

| ツール | 主な用途 | 痛み | Matlens でカバー |
|---|---|---|---|
| Excel | 試験条件 / 結果記録 / 集計 | 散在、検索性低、版管理なし | ✅ 構造化データ + 検索 |
| Word / PowerPoint | 報告書 / 提案資料 | テンプレ崩れ / 版管理 | △ Markdown 本文（PDF 化は Phase 4）|
| OneDrive / SharePoint | 文書共有 | 検索性低、命名規則依存 | △ 横断セマンティック検索 |
| Outlook / Teams | 連絡 / 進捗共有 | 情報が会話に埋もれる | ✅ 活動タイムラインで見える化 |
| ホワイトボード | 進捗管理 | リモートで見えない、写真撮影必要 | ✅ Specimen Tracker Kanban |
| 紙台帳 | 受入記録 / 工具履歴 | 検索不可、紛失リスク | ✅ 試験片 / 工具マスタ |

### 切削プロセス系ツール

| ツール | 主な用途 | Matlens との位置 |
|---|---|---|
| **CutPro** (MAL) | Stability Lobe / 切削力解析 | 補完（Matlens は俯瞰、CutPro は深掘り）|
| **Sandvik / Kennametal カタログ** | 工具選定 | 補完（社内蓄積を Matlens に集約）|
| **TopSolid'CAM / hyperMILL** | CAM 軌跡生成 | 連携（Matlens は条件参照）|
| LabVIEW + センサ | 切削抵抗・振動データ取得 | 連携（Matlens は波形保存）|
| MATLAB | 波形 FFT / 統計 | 補完（Matlens は可視化、深い解析は MATLAB）|

### CAE / 解析ソフト

| ツール | 用途 | 連携可能性 |
|---|---|---|
| ANSYS / ABAQUS | 構造・熱解析 | Matlens の材料物性値を CAE 入力に渡す |
| LS-DYNA | 衝突 / 加工解析 | 同上 |
| Deform | 鍛造 / 機械加工解析 | 切削条件を Matlens から渡す |

## Matlens がフィットする領域マトリクス（仮説）

| 領域 | 既存ツールの強み | Matlens の強み | 推奨ポジション |
|---|---|---|---|
| 個別試験データの厳密管理 | LIMS が圧倒的 | 弱い（ない） | 補完 / 連携 |
| 過去事例の俯瞰検索 | LIMS は機能少 | 強い | **置換候補** |
| 切削プロセス可視化 | 専用ツールあり | UI 統合度 | 補完 |
| ドメイン横断のスクラッチパッド | 各ツールに分散 | 一画面 | **新領域** |
| 顧客向けレポート | Word が王道 | Markdown PoC | 連携 |
| 業務 KPI ダッシュボード | LIMS が一部対応 | 軽量・即見えに強い | **新領域** |

→ ヒアリング後、「補完」「連携」「置換」「新領域」のどれで攻めるかを案件ごとに決める。

## 移行 / 共存パターンの想定

| パターン | 対象 | 期間 | リスク |
|---|---|---|---|
| **連携レイヤ** | 既存 LIMS が強固 | PoC 4 ヶ月で API 連携設計 | LIMS 側の API 開放可否 |
| **段階置換** | 改修の遅い社内 LIMS | 1〜2 年で部分置換 | 業務移行ストレス、データ移行コスト |
| **新領域追加** | 切削プロセス管理（既存ツール薄）| 4 ヶ月で MVP | ドメイン理解不足、要件発散 |
| **シャドー運用** | 様子見フェーズ | 6 ヶ月並列、月次検証 | 二重入力負荷、定着率低下 |

ヒアリングで「どのパターンが現実的か」を引き出すことが、PoC 後の本格導入提案につながる。

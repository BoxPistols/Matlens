# ADR-006: 試験片トラッカーを Kanban + Table の二重ビューで提供する

- ステータス: Accepted
- 日付: 2026-04-18
- 関連 issue / PR: #45 (Phase 3 試験片トラッカー)

## 背景

試験片トラッカー (`#/specimens`) は、受入〜試験〜保管〜廃棄のライフサイクルを
追う画面。ヒアリング前提で想定されるペルソナが 2 つある。

- **試験技師（オペレータ）**: 「今自分が持ってる試験片」「次に試験する試験片」を
  視覚的に把握したい → Kanban 向き
- **PM / 研究員**: 「納期近い全試験片を一覧で見たい」「材料別に絞って比較」など
  データ的な操作 → Table 向き

片方だけだと、

- Kanban のみ: 全案件横断で数値を見るのに不向き、期日ソート・フィルタが難しい
- Table のみ: 物理的なフロー感覚が失われ、「今どこにある？」が直感的でない

## 決定

**同一画面に Kanban / Table のビュー切替ボタン** を置く。

- 初期値はユーザ設定 (`localStorage`) に依存。未設定時は Kanban
- 両ビューで同じフィルタ・検索 UI を共有
- 同じ Repository を参照（`specimensRepository.list()`）し、表示のみ切替
- D&D によるステータス遷移は Kanban 側のみ（Phase 4 候補）
- Table 側は並び替え・CSV 書き出しを提供

## 代替案と棄却理由

| 案 | 棄却理由 |
|---|---|
| Kanban のみ | PM / 研究員の横断ニーズに合わない。100 件超の試験片で視認性崩壊 |
| Table のみ | 試験技師の物理フロー把握が不便 |
| 別画面として分ける | フィルタ状態の同期コストが過大、メニューが肥大化 |
| Scrum ボード的な Kanban + 別タブで Table | タブ切替は 1 クリックだが、同画面ボタンでも 1 クリックで
  状態共有が自然になる利点あり |

## 実装ポイント

- ビュー切替は `useSpecimenViewMode()` フック（`localStorage` + state）
- Kanban の列順は固定（受入 → 試験前 → 試験中 → 試験後 → 保管 → 廃棄）
- 列のスクロールは水平、カードは縦スクロール
- Table は `ProjectsTable` と同じ列コンポーネントを再利用
- ペルソナ別プリセット（将来）:
  - 「試験技師プリセット」→ Kanban + 自担当フィルタ
  - 「PM プリセット」→ Table + 納期 7 日以内フィルタ

## 影響

### ポジティブ
- 1 画面で 2 ペルソナをカバーできる
- フィルタ実装が DRY
- 現場ヒアリングで「どっち派？」の質問が自然に生まれる
  → デフォルトビューをペルソナ別に出し分ける改善に繋がる

### ネガティブ
- 両ビューで UI テスト・スナップショットが 2 倍
- モバイル幅で Kanban の横スクロールが厳しい
  → 狭幅では Table 強制の分岐を検討（Phase 4）

## 将来の検討事項

- D&D でのステータス遷移（楽観的更新 + ロールバック）
- 保管場所のロケーション QR コード連携
- 試験片個体の写真・寸法データの添付（損傷ギャラリーとのクロス表示）

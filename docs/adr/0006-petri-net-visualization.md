# ADR 0006 — ペトリネット可視化ライブラリ選定

| 項目 | 内容 |
|---|---|
| 状態 | 採用 |
| 決定日 | 2026-04-12 |
| 関連 issue | #17 (Phase B) |

## 背景

金属試験ワークフロー（計画 → 材料選定 → 加工 → 試験 → レポート）を可視化する際、
工程間のフィードバックループ（再加工: 後加工済 → 一次加工済）が存在する。

DAG (有向非巡回グラフ) ではサイクルを表現できないため、ペトリネット (P/T net) を採用した。

## 検討した選択肢

### A. `@xyflow/react` + `@dagrejs/dagre`（却下）

- **メリット**: ドラッグ&ドロップ、ズーム、自動レイアウト、エコシステムが豊富
- **デメリット**: gzip 約 150 kB の追加バンドル。
  現在のメインバンドル 122 kB に対して不釣り合い。
  lazy load で分割できるが、初回表示が遅くなる。
  また dagre はメンテナンスが散漫（@dagrejs/dagre フォークを使う必要あり）。

### B. 純 SVG 自作（採用）

- **メリット**:
  - 追加依存ゼロ（バンドルサイズ影響なし）
  - ペトリネット固有のセマンティクス（Place/Transition/Token）を型安全に実装できる
  - PNML エクスポートと直接連動
  - CSS カスタムプロパティでテーマ対応
- **デメリット**:
  - ドラッグ&ドロップ・ズームは手動実装が必要（MVP スコープ外）
  - レイアウトは固定（3 行スネークパターン）

## 決定

**純 SVG 自作** を採用する。

MVP では固定レイアウト（3 行スネーク + 上方フィードバック弧 + 下方廃棄ブランチ）で、
トークンのインタラクティブな発火操作と PNML エクスポートを提供する。

将来的にドラッグ操作やより複雑なネットが必要になった場合に `@xyflow/react` へ移行する。

## レイアウト設計

```
Row 1 (y=90): p0 → t0 → p1 → t1 → p2 → t2 → p3 → t3 → p4
                                              ↑t4(再加工弧)↗
Row 2 (y=220):         t5 ← p5 ← t6 ← p6 ← t7 ← p7 ← t8 ← p8 ← t9
                         ↑(p4より)         ↓t11(廃棄)
Row 3 (y=350):         p9 → t10 → p10      p11(廃棄)
```

- `t4`（再加工）は Row 1 上方の cubic bezier 弧で表現。DAG 非対応の理由を視覚的に示す。
- `t11`（廃棄判定）は `p6`（評価済）から Row 3 へのブランチ。

## PNML エクスポート

`src/services/pnml.ts` が ISO/IEC 15909-2 準拠の PNML XML を生成する。
PIPE・GreatSPN・CPN Tools 等の外部ツールで読み込み・解析可能。
